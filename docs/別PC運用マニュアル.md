# 別PCで「せどり新着アラート」を使うためのマニュアル

このマニュアルは、今作っている `sedori-alert` を別のWindowsパソコンで使うための手順です。

## まず大事な考え方

GitHubには、主に「ツール本体」が保存されています。

GitHubに保存されるもの:

- 画面
- 通知の仕組み
- 商品取得の仕組み
- キーワード判定の仕組み
- 起動用ファイル
- 開発履歴

GitHubに保存しないもの:

- Discord Webhook URL
- 登録済みキーワード
- 通知履歴
- 商品取得履歴
- ローカルデータベース
- 個人用の秘密情報

そのため、別PCでは基本的に「ツール本体をGitHubから入れる」あとに、Discord通知やキーワードをもう一度設定します。

## 全体の流れ

1. 別PCに必要なアプリを入れる
2. GitHub Desktopでツール本体をコピーする
3. ツールを初回起動する
4. ログインする
5. Discord通知を設定する
6. キーワードを登録する
7. 通知テストをする
8. 運用開始する

## 1. 別PCに入れるもの

最初に、別PCへ以下を入れてください。

### 必須

- GitHub Desktop
- Python
- Node.js
- Google Chrome または Microsoft Edge

### 入れ方の目安

GitHub Desktop:

- GitHubからツール本体をコピーするために使います。

Python:

- 裏側の処理を動かすために使います。
- インストール時に `Add python.exe to PATH` のようなチェックが出たら、チェックを入れてください。

Node.js:

- 画面側を準備するために使います。
- LTS版で大丈夫です。

Chrome または Edge:

- セカンドストリートなど、一部サイト取得の補助に使います。

## 2. GitHub Desktopでツール本体をコピーする

別PCでGitHub Desktopを開きます。

1. GitHub Desktopを開く
2. 上のメニューから `File` を押す
3. `Clone repository...` を押す
4. 一覧から `sedori-alert` を選ぶ
5. 保存先を選ぶ
6. `Clone` を押す

もし一覧に出ない場合:

1. `URL` タブを開く
2. 次のURLを入れる

```text
https://github.com/ariyya0628-rgb/sedori-alert
```

3. 保存先を選ぶ
4. `Clone` を押す

おすすめの保存先:

```text
C:\Users\自分のユーザー名\Documents\Codex\sedori-alert
```

保存先は別の場所でも動きますが、あとで見つけやすい場所がおすすめです。

## 3. フォルダを確認する

コピーが終わったら、フォルダの中に以下があるか確認してください。

```text
backend
frontend
scripts
00_START.bat
README.md
```

`00_START.bat` が起動用ファイルです。

## 4. 初回起動する

`00_START.bat` をダブルクリックします。

黒いPowerShell画面が開きます。

初回だけ、必要な部品を自動で入れるため時間がかかります。

目安:

- 早いPC: 数分
- 遅いPC: 10分以上かかることもあります

黒い画面は閉じないでください。

正常に起動すると、ブラウザで以下が開きます。

```text
http://127.0.0.1:8000/
```

もし自動で開かない場合は、ブラウザに上のURLを直接入れてください。

## 5. ログインする

初期ログイン情報:

```text
メールアドレス: admin@example.com
パスワード: password123
```

このツールは今のところ自分専用のローカル利用前提です。
外部に公開する用途では使わないでください。

## 6. Discord通知を設定する

別PCではDiscord通知設定をもう一度入れる必要があります。

手順:

1. Discordを開く
2. 通知を飛ばしたいサーバーを開く
3. サーバー設定を開く
4. `連携サービス` を開く
5. `ウェブフック` を開く
6. `新しいウェブフック` を作る
7. 投稿先チャンネルを選ぶ
8. `ウェブフックURLをコピー` を押す
9. せどり新着アラートの `Discord設定` に貼り付ける
10. `保存` を押す
11. `テスト通知` を押す

Discordにテスト通知が来ればOKです。

## 7. キーワードを登録する

`キーワード管理` を開いて登録します。

基本項目:

- キーワード
- ショップ
- 価格下限
- 価格上限
- 表記ゆれ / 追加一致語
- 除外語

状態条件:

- セカスト通知ランク
- オフモール通知ランク
- まんだらけ通知ランク
- 駿河屋通知状態

### 入力例

レコルトを探す例:

```text
キーワード: レコルト
表記ゆれ / 追加一致語:
recolte
RRF

除外語:
ジャンク
部品
```

価格条件の例:

```text
価格下限: 1000
価格上限: 8000
```

空欄にすると、その条件は使われません。

## 8. 店舗別の状態ランク設定

状態ランクはショップごとに基準が違うため、別々に設定します。

### セカンドストリート

入力例:

```text
新品,未使用品,中古A,中古B
```

意味:

- 新品
- 未使用品
- 中古A
- 中古B
- 中古C
- 中古D

通知したくないランクは入れないでください。

### オフモール

入力例:

```text
N,S,A,B
```

意味:

- N: 新品・未開封現行品
- S: 新品同様・廃盤商品
- A: 使用回数の少ない美品
- B: 日常使用の中古・傷汚少
- C: 使用感ある中古・傷汚れ多
- D: 傷汚れ剥げなどダメージ多

### まんだらけ

入力例:

```text
10,9,8,7
```

意味:

- 10: 美
- 9: 美から並上
- 8: 並上から並
- 7: 並
- 6: 並から並下
- 5: 並から並下
- 4: 並下
- 3: 難あり
- 2: 現状品
- 1: 現状品

### 駿河屋

駿河屋は明確なランク表記が少ないため、商品名などに含まれる言葉から分類します。

おすすめ入力:

```text
未開封品,帯付き,美品,通常中古
```

分類:

- 未開封品
- 帯付き
- 美品
- 通常中古
- 注意あり
- ジャンク

`注意あり` や `ジャンク` を通知したくない場合は、入力欄に入れないでください。

## 9. 商品取得テスト

ダッシュボードの `実ショップ取得テスト` で確認します。

おすすめの確認:

```text
ショップ: オフモール
キーワード: レコルト
件数: 5
```

成功すると、取得数が表示されます。

次に、商品一覧に商品が入っているか確認します。

## 10. Discord通知の確認

確認する順番:

1. `Discord設定` でテスト通知を送る
2. Discordに届くか見る
3. キーワードを登録する
4. 実ショップ取得テストをする
5. 商品一覧と通知履歴を見る
6. 条件に合う商品があればDiscord通知が届くか見る

通知履歴には以下が残ります。

- 通知した理由
- 通知しなかった理由

例:

```text
一致語: recolte
価格が上限超過: 9,000円 > 8,000円
セカスト状態対象外: 中古C
除外語に一致: ジャンク
```

## 11. 運用中の起動方法

毎回使うときは、基本的にこれだけです。

```text
00_START.bat
```

黒いPowerShell画面は閉じないでください。

閉じるとツールも止まります。

## 12. PCの電源について

このツールは別PC上で動くため、そのPCの電源が切れている間は動きません。

通知を受けたい時間帯は、以下が必要です。

- PCの電源が入っている
- Windowsにログインしている
- インターネットにつながっている
- ツールが起動している
- 黒いPowerShell画面を閉じていない

## 13. 自動巡回について

画面を開かずに登録済みキーワードを巡回したい場合は、以下の仕組みがあります。

```text
scripts\start-keyword-monitor.ps1
```

Windowsログイン時に自動で巡回を始めたい場合:

```text
scripts\install-keyword-monitor-task.ps1
```

ただし、別PCの初回運用では、まず手動起動で数日様子を見るのがおすすめです。
通知が安定してから自動巡回に切り替えてください。

## 14. GitHub Desktopで最新版を取り込む

今後、私がこのPC側で修正してGitHubに保存した場合、別PCでは最新版を取り込む必要があります。

別PCでの操作:

1. GitHub Desktopを開く
2. `Current repository` が `sedori-alert` になっているか確認
3. 上部に `Fetch origin` が出ていたら押す
4. `Pull origin` が出たら押す

意味:

- `Fetch origin`: GitHubに新しい更新があるか確認
- `Pull origin`: GitHubの最新版をこのPCに取り込む

普段は、別PCでは `Pull origin` を押す側になります。

## 15. 別PCで変更した場合

別PC側でも私が修正作業をした場合は、GitHub Desktopで `Push origin` を押します。

意味:

- `Push origin`: このPCの変更をGitHubに保存する

ただし、普段使うだけなら `Push origin` はあまり押しません。
コード修正をしたときだけです。

## 16. よくあるトラブル

### 画面が開かない

確認すること:

- `00_START.bat` を起動しているか
- 黒いPowerShell画面が開いたままか
- エラーが表示されていないか
- ブラウザで `http://127.0.0.1:8000/` を開いているか

### 黒い画面がすぐ閉じる

PythonかNode.jsが入っていない可能性があります。

その場合は、PythonとNode.jsを入れたあと、もう一度 `00_START.bat` を開いてください。

### Discord通知が来ない

確認すること:

- Discord設定にWebhook URLを貼ったか
- `通知を有効にする` がオンか
- `保存` を押したか
- `テスト通知` が届くか
- キーワード条件が厳しすぎないか
- 通知履歴に「通知しなかった理由」が出ていないか

### 商品はあるのに通知されない

通知履歴を見てください。

よくある理由:

- 価格上限を超えている
- 価格下限より安い
- 除外語に一致している
- 状態ランクが対象外
- すでに通知済みで重複扱いになっている

### セカストだけ取得に失敗する

セカストは取得制限が出ることがあります。

確認すること:

- ChromeまたはEdgeが入っているか
- PCのネット接続が安定しているか
- 少し時間を空けて再実行する

## 17. 別PCへ移さないほうがいいもの

以下はGitHubに上げたり、人に送ったりしないでください。

```text
backend\sedori_alert.db
backend\.venv
frontend\node_modules
frontend\dist
logs
```

特に `backend\sedori_alert.db` には、設定や履歴が入ります。

## 18. 別PCへ履歴やキーワードも移したい場合

基本は別PCで再設定がおすすめです。

どうしても移したい場合は、以下のファイルをコピーする方法があります。

```text
backend\sedori_alert.db
```

ただし、このファイルにはDiscord設定なども含まれるため、扱いには注意してください。
移行する場合は、事前にバックアップを取ってから行ってください。

## 19. 最初にやるおすすめチェックリスト

別PCで最初にやること:

- GitHub Desktopで `sedori-alert` をCloneした
- `00_START.bat` を起動した
- `http://127.0.0.1:8000/` が開いた
- `admin@example.com` / `password123` でログインできた
- Discord Webhook URLを設定した
- テスト通知が届いた
- キーワードを1つ登録した
- 実ショップ取得テストが成功した
- 商品一覧に商品が出た
- 通知履歴を確認した

ここまでできれば、別PCでの初期運用は開始できます。
